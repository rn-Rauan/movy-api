# Subscriptions Module

Manages the lifecycle of organisation subscriptions to plans on the Movy platform.

---

## Responsibilities

- Subscribe an organisation to a plan (creates a 30-day subscription)
- Cancel an existing subscription
- Query the active subscription for an organisation
- List all subscriptions for an organisation (paginated)

## Domain Entity — `SubscriptionEntity`

| Field            | Type                 | Notes                                        |
|------------------|----------------------|----------------------------------------------|
| `id`             | `string` (UUID)      | Generated by `crypto.randomUUID()` in domain |
| `organizationId` | `string` (UUID)      | Immutable after creation                     |
| `planId`         | `number`             | FK to `Plan.id`; immutable after creation    |
| `status`         | `SubscriptionStatus` | Starts as `ACTIVE`; can be `CANCELED`        |
| `startDate`      | `Date`               | Set to `now` at creation; immutable          |
| `expiresAt`      | `Date`               | Set to `startDate + 30 days` by the use case |
| `createdAt`      | `Date`               | Immutable                                    |
| `updatedAt`      | `Date`               | Refreshed by `cancel()`                      |

### Key methods

| Method           | Description                                                  |
|------------------|--------------------------------------------------------------|
| `create(props)`  | Factory — creates a new `ACTIVE` subscription with a UUID    |
| `restore(props)` | Mapper-only factory — reconstructs entity from DB record     |
| `cancel()`       | Sets `status = CANCELED`, updates `updatedAt`               |

## Use Cases

### `SubscribeToPlanUseCase`

```
POST /organizations/:organizationId/subscriptions
```

1. Validates that the plan exists and `isActive = true`
2. Checks that the organisation has no current `ACTIVE` subscription
3. Calculates `expiresAt = now + 30 days`
4. Creates and persists the `SubscriptionEntity`

**Throws:**
- `PlanNotFoundError` — plan does not exist or is inactive
- `SubscriptionAlreadyActiveError` — organisation already subscribed
- `SubscriptionCreationFailedError` — repository persistence failure

---

### `CancelSubscriptionUseCase`

```
PATCH /organizations/:organizationId/subscriptions/:id/cancel
```

1. Looks up the subscription by UUID
2. Verifies `subscription.organizationId === organizationId` (tenant check)
3. Calls `subscription.cancel()`
4. Persists only `status` + `updatedAt` (partial update)

**Throws:**
- `SubscriptionNotFoundError` — subscription not found
- `SubscriptionForbiddenError` — subscription belongs to another organisation

---

### `FindActiveSubscriptionUseCase`

```
GET /organizations/:organizationId/subscriptions/active
```

Returns the most recent `ACTIVE` subscription for the organisation, or `null` if none exists.

---

### `FindSubscriptionsByOrganizationUseCase`

```
GET /organizations/:organizationId/subscriptions?page=1&limit=10
```

Returns a paginated list of all subscriptions (any status) for the organisation.

## Subscription Status

| Value      | Description                                                        |
|------------|--------------------------------------------------------------------|
| `ACTIVE`   | Valid subscription; grants plan benefits                           |
| `CANCELED` | Explicitly cancelled by an admin                                   |
| `PAST_DUE` | Expired without renewal (set by external scheduler or webhook)     |

## Guards & Access Control

All endpoints are protected by:
- `JwtAuthGuard` — requires a valid JWT
- `TenantFilterGuard` — scopes the request to the authenticated user's organisation
- `RolesGuard` + `@Roles(RoleName.ADMIN)` — restricts access to organisation administrators
